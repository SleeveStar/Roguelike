# GitHub 연동 순서

이 문서는 로컬 프로젝트를 GitHub 저장소와 연동하는 과정을 안내합니다.

## 1단계: 로컬 Git 저장소 초기화
아직 Git 저장소가 초기화되지 않았다면, 프로젝트의 루트 디렉토리에서 다음 명령어를 실행합니다.
```bash
git init
```

## 2단계: 파일 스테이징
현재 프로젝트의 모든 변경 사항을 Git에 추가합니다.
```bash
git add .
```
특정 파일만 추가하고 싶다면 `.` 대신 파일 경로를 사용합니다. 예: `git add src/main/resources/static/js/constants.js`

## 3단계: 변경 사항 커밋
스테이징된 변경 사항들을 커밋합니다. 커밋 메시지는 변경 내용을 잘 설명해야 합니다.
```bash
git commit -m "첫 번째 커밋: 프로젝트 초기 설정 및 기본 파일 추가"
```
또는 더 자세한 메시지를 원하면:
```bash
git commit -m "feat: 초기 프로젝트 설정 완료" -m "모든 기본 파일과 게임 로직, UI 개선 사항이 포함된 초기 커밋입니다."
```

## 4단계: GitHub에 새 저장소 생성
GitHub 웹사이트(github.com)로 이동하여 새 저장소(New repository)를 생성합니다.
*   **저장소 이름**을 지정합니다. (예: `webroguelike`)
*   **공개(Public)** 또는 **비공개(Private)** 여부를 선택합니다.
*   **README, .gitignore, 라이선스** 파일은 *현재 생성하지 않습니다*. (로컬 프로젝트의 파일을 사용할 것입니다.)
저장소 생성 후, GitHub에서 제공하는 "…or push an existing repository from the command line" 섹션의 원격 저장소 URL을 복사합니다. (예: `https://github.com/YOUR_USERNAME/YOUR_REPOSITORY_NAME.git`)

## 5단계: 원격 저장소 추가
로컬 Git 저장소에 GitHub에 생성한 원격 저장소를 연결합니다. `YOUR_REPOSITORY_URL`을 4단계에서 복사한 URL로 교체합니다.
```bash
git remote add origin YOUR_REPOSITORY_URL
```
예시:
```bash
git remote add origin https://github.com/your-username/webroguelike.git
```
기존 `origin`이 있다면 `git remote set-url origin YOUR_REPOSITORY_URL`을 사용하거나, `git remote -v`로 확인 후 `git remote rm origin`으로 삭제 후 다시 추가할 수 있습니다.

## 6단계: 변경 사항 푸시
로컬 커밋을 GitHub의 원격 저장소로 푸시합니다.
```bash
git push -u origin main
```
만약 기본 브랜치 이름이 `master`라면 `main` 대신 `master`를 사용합니다.
(처음 푸시할 때는 `-u origin main`을 사용하여 로컬 `main` 브랜치가 원격 `origin`의 `main` 브랜치를 추적하도록 설정합니다. 이후부터는 `git push`만 사용해도 됩니다.)

## 완료!
이제 로컬 프로젝트가 GitHub 저장소에 성공적으로 업로드되었으며, 변경 사항을 계속 추적하고 공유할 수 있습니다.
